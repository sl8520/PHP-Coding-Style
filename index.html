PHP Coding Style

PHP標籤

PHP 必須 使用長標籤 <?php ?> 或 <?= ?> 短輸出標籤， 不可 使用其他自定義標籤。【PSR-1】

文件

  • PHP文件 必須 使用 Unix LF (linefeed) 作為行的結數符【PSR-2】
  • PHP文件 必須 以一個空白行作為結束。【PSR-2】
  • 純PHP文件 必須省 略最後的 ?> 標籤。【PSR-2】

文件命名

  • 小寫駝峰式命名法（camelCase），小寫字母開頭，後面每個單字的第一個字母大寫。
  • 檔名長度必須小於 32 個字元。
  • 使用 .php 作為副檔名。
  • 會被引入的設定檔以 .inc.php 作為結尾。

字元編碼

PHP 必須 使用無BOM的UTF-8編碼。【PSR-1】

字串

字串 必須 使用單引號，3 個例外

  • \n
  • 字串內容包含單引號
  • SQL字串

縮排

程式碼 必須 使用4個space不使用tab縮進。【PSR-2】

行

  • 每行的字符數建議在80以內，最多不超過120字，但無硬性限制。【PSR-2】
  • 非空白行後面 不可 有多餘的空白符。【PSR-2】

關鍵字 及 true/false/null

  • PHP所有關鍵字 必須 全部小寫。【PSR-2】
  • SQL所有關鍵字 必須 全部大寫。
  • 常數 true、false、null也 必須 全部小寫。【PSR-2】

變數命名

  • 小寫駝峰式命名法（camelCase），小寫字母開頭，後面每個單字的第一個字母大寫。
  • 迴圈變數通常使用 $i、 $j、$k，依此類推。
  • 使用描述性的命名方式。
  • 盡量不要縮寫，把完整單字寫出，除非有特定領域大家都可接受的縮寫，以確保其他人一看就看得懂。
  • boolean 變數的開頭 必須 使用is、can、has、should，但若本身形容詞就不需要，如 enabled、done。
  • 使用介系詞 ( of、for、from、on… ) 來組合變數名稱。
  • e.g.
<?php

$daysSinceModification = 3; // 修改已過了3天
$workDaysPerWeek = 5; // 每週工作5天
$daysUntilDeadline = 10; // deadline前還剩10天

常數命名

  • 常數名稱完全採用大寫英文字母。【PSR-1】
  • 兩個單字以上時用底線 _ 來區隔。【PSR-1】

Array

// 索引陣列
$ary = array('apple', 'banana');
$ary = array(
    'apple',
    'banana'
);

// Laravel、Lumen
$ary = ['apple', 'banana'];
$ary = [
    'apple',
    'banana'
];

// 關聯陣列
$ary = array('one' => 'apple', 'two' => 'banana');
$ary = array(
    'one' => 'apple',
    'two' => 'banana'
);

// Laravel、Lumen
$ary = ['one' => 'apple', 'two' => 'banana'];
$ary = [
    'one' => 'apple',
    'two' => 'banana'
];
  • 專案為 Laravel、Lumen 時 一律 使用中括號縮寫。
  • 元素的 , 之前 不可 有空白，而 , 之後 必須 留一個空白。
  • 關聯陣列的 => 前後 必須 有空白。
  • 元素分成多行
    • 第一個元素 必須 在新的一行，且每一行只有一個元素。
    • 元素的結束 ) 必須 在新的一行。
  • 若元素低於 5 個時可縮成一行。

Namespace

<?php

namespace Vendor\Package;

use FooClass;
use BarClass as Bar;
use OtherVendor\OtherPackage\BazClass;

class ClassName 
{
    // class body
}
  • 大寫駝峰式命名法（Pascal Case），大寫字母開頭，後面每個單字的第一個字母大寫。
  • 所有 use 必須 在 namespace 後聲明。【PSR-2】
  • 每條 use 語句 必須 只有一個 use 關鍵字。【PSR-2】
  • <?php 與 namespace 之間 必須 空一行。【PSR-2】
  • namespace 與 use 之間 必須 空一行。【PSR-2】
  • use 與 class 之間 必須 空一行。【PSR-2】

Interface

<?php

interface PostRepositoryInterface
{
    public function getLatest3Posts();
}
  • 大寫駝峰式命名法（Pascal Case），大寫字母開頭，後面每個單字的第一個字母大寫。
  • 一個 interface 一個檔案。【PSR-2】
  • 括號 { 必須 換行。【PSR-2】

Class

<?php

namespace Vendor\Package;

use FooClass;
use BarClass as Bar;
use OtherVendor\OtherPackage\BazClass;

class ClassName extends ParentClass implements \ArrayAccess, \Countable
{
    // class body
}
  • 大寫駝峰式命名法（Pascal Case），大寫字母開頭，後面每個單字的第一個字母大寫。
  • 一個 class 一個檔案。【PSR-2】
  • 括號 { 必須 換行。【PSR-2】
  • extends 與 implements 必須 與 class同一行。【PSR-2】
  • implements 的繼承列可以分成多行，每個 interface 都 必須 分開獨立成一行，包含第一個。【PSR-2】

Trait

<?php

trait LogTrait
{
    public function startLog()
    {
        // method body
    }

    public function stopLog()
    {
        // method body
    }
}
  • 大寫駝峰式命名法（Pascal Case），大寫字母開頭，後面每個單字的第一個字母大寫。
  • 一個 trait 一個檔案。【PSR-2】
  • 括號 { 必須 換行。【PSR-2】

Property

<?php

class ClassName
{
    public $userName;
    public static $userCountry;
    private $_userPassword;
    protected $_userAddr;
}
  • 小寫駝峰式命名法（camelCase），小寫字母開頭，後面每個單字的第一個字母大寫。
  • 每個屬性都 必須 加上訪問修飾詞 ( public、protected、private )。【PSR-2】
  • static 必須加在訪問修飾詞之後。
  • 每條語句 不可 定義超過一個屬性。【PSR-2】
  • 不可 再使用var宣告 public property。【PSR-2】
  • 使用底線 _ 來宣告屬性為 private 與 protected。[1]

Method

<?php

namespace Vendor\Package;

class ClassName
{
    public function fooBar($arg1, &$arg2, array $arg3 = [])
    {
        // method body
    }
    
    public function multiLineArgument(
        $arg1,
        &$arg2,
        array $arg3 = []
    ) {
       // method body
    }
}
  • 小寫駝峰式命名法（camelCase），小寫字母開頭，後面每個單字的第一個字母大寫。
  • 括號 { 必須 換行。【PSR-2】
  • 每個方法都 必須 加上訪問修飾詞 ( public、protected、private )。【PSR-2】
  • 方法名稱與 ( 之間 不可 留空白。【PSR-2】
  • 第一個 ( 之後 不可 有空白，最後一個 ) 之前 不可 有空白。【PSR-2】
  • 參數的 , 之前 不可 有空白，而 , 之後 必須 留一個空白。【PSR-2】
  • 參數列若提供 預設值，則 必須 放在最後一個參數。【PSR-2】
  • 參數列可以分成多行
    • 第一個參數 必須 在新的一行，且每一行只有一個參數。【PSR-2】
    • 參數列的結束 ) 必須 在新的一行，且與 ) 與 { 同行，之間 必須 留一個空白。【PSR-2】
  • 使用底線 _ 來宣告方法為 private 與 protected。[1:1]
  • 命名前綴使用動詞。可參考 微軟 PowerShell 指令動詞列表

abstract、final 及 static

<?php

namespace Vendor\Package;

abstract class ClassName
{
    protected static $_foo;

    abstract protected function _zim();

    final public static function bar()
    {
        // method body
    }
}
  • abstract 與 final 必須 加在訪問修飾詞之前。【PSR-2】
  • static 必須 加在訪問修飾詞之後。【PSR-2】

Method and Function Calls

<?php

bar();
$foo->bar($arg1);
Foo::bar($arg2, $arg3);

$foo->bar(
    $arg1,
    $arg2,
    $arg3
);
  • 方法或函數名稱與 ( 之間 不可 有空白。【PSR-2】
  • 第一個 ( 之後 不可 有空白，最後一個 ) 之前 不可 有空白。【PSR-2】
  • 引數的 , 之前 不可 有空白，而 , 之後 必須 留一個空白。【PSR-2】
  • 引數列可以分成多行
    • 第一個引數 必須 在新的一行，且每一行只有一個引數。【PSR-2】
    • 引數列的結束 ) 必須 在新的一行。【PSR-2】

Closure

<?php

$closureWithArgs = function ($arg1, $arg2) {
    // body
};

$closureWithArgsAndVars = function ($arg1, $arg2) use ($var1, $var2) {
    // body
};
  • function 與 ( 之間 必須 留一個空白。【PSR-2】
  • use 前後都 必須 留一個空白。【PSR-2】
  • { 必須 與 function 同一行。【PSR-2】
  • 第一個 ( 之後 不可 有空白，最後一個 ) 之前 不可 有空白。【PSR-2】
  • 參數的 , 之前 不可 有空白，而 , 之後 必須 留一個空白。【PSR-2】
  • 參數列若提供 預設值，則 必須 放在最後一個參數。【PSR-2】

控制結構【PSR-2】

控制結構的基本規範如下：

  • 控制結構關鍵詞後 必須 有一個空格。
  • 左括號 ( 後 不可 有空格。
  • 右括號 ) 前也 不可 有空格。
  • 右括號 ) 與開始大括號右括號 {間 必須 有一個空格。
  • 結構體主體 必須 要有一次的縮進。
  • 結束大括號 } 必須 在結構體主體後單獨成新的一行。

if 、 else if 和 else

<?php

if ($expr1) {
    // if body
} elseif ($expr2) {
    // elseif body
} else {
    // else body;
}
  • { 與 if 、 else if 、 else 要在同一行，不用換行。
  • } 與 else if 要在同一行，不用換行。
  • } 與 else 要在同一行，不用換行。

switch 和 case

<?php

switch ($expr) {
    case 0:
        echo 'First case, with a break';
        break;
    case 1:
        echo 'Second case, which falls through';
        // no break
    case 2:
    case 3:
    case 4:
        echo 'Third case, return instead of break';
        return;
    default:
        echo 'Default case';
        break;
}
  • { 與 switch 在同一行。
  • case 後的條件與 : 之間不可以有空白。
  • case 必須縮排於 switch，而 break 必須縮排於case。
  • 如果存在 case 內沒有 break ，主體裡需加上 // no break 註解。

while 和 do while

<?php

while ($expr) {
    // structure body
}

{ 與 while 在同一行。

<?php

do {
    // structure body;
} while ($expr);
  • do與 { 在同一行。
  • } 與 while 在同一行。

for

<?php

for ($i = 0; $i < 10; $i++) {
    // for body
}
  • { 與 for 在同一行。
  • =、< 或 > 前後要留一個空白。
  • ; 之前 不可 有空白，之後 必須 留一個空白。
  • 第一個 ( 之後 不可 有空白，最後一個 ) 之前 不可 有空白。

foreach

<?php

foreach ($iterable as $key => $value) {
    // foreach body
}
  • { 與 foreach 在同一行。
  • => 前後要留一個空白。
  • 第一個 ( 之後 不可以有空白，最後一個 ) 之前 不可以有空白。

try, catch

<?php

try {
    // try body
} catch (FirstExceptionType $e) {
    // catch body
} catch (OtherExceptionType $e) {
    // catch body
} finally {
    // finally body
}
  • { 與try, catch 要在同一行，不用換行。
  • } 與 catch 要在同一行，不用換行。
  • } 與 finally 要在同一行，不用換行。
